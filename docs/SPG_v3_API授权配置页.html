SPG v3 Setup

API 授权与配置中枢

系统第一次启动不直接进入星河主界面，而是先进入这个授权页。你在这里配置 Provider Key、月预算、默认角色、回退策略，并完成连接测试。只有通过核心检查，主系统才解锁。

Setup Flow
Step 01 基础设定 项目环境、默认工作区、配置存储位置、加密方式。
Step 02 Provider 授权 填写 API Key、Base URL、预算和默认角色。
Step 03 连接测试 逐个验证可用性、响应时间、额度读取情况。
Step 04 调度策略 确认谁是主入口、谁做精查、谁做 fallback。
Step 05 解锁主系统 核心链路通过后，才开放星河主工作台。

Provider Settings

这不是给开发者看的隐藏配置，而是正式产品里对外可见的授权页。你以后提供 API 信息时，应该在这里配置，而不是改代码。

2 / 6 Providers Ready

豆包 API

日常主解析器。负责摘要、实体归并、事件归因、结构化信号输出。因为无限额，所以作为默认 Reason 层主力。

Core / Ready

Serper API

主搜索入口。默认承担候选发现与关键词扩展。由于月预算最高，应成为主搜索 provider。

Core / Ready

Firecrawl API

高价值页面正文抽取器。只给高分页面和官方源，不应成为全量采集入口。

Core / Pending

Tavily API

时效验证入口。适合争议、突发事件、新闻型信号的真实性和扩散判断。

Primary Verify

Exa API

长尾与语义扩展能力。用于难搜、冷门站点、语义相关结果的补足，不进入默认热路径。

Optional

Gemini API

复杂 fallback 与多模态解释能力。不是默认常用 provider，但适合特殊页面和异常内容。

Optional

解锁条件

这不是“配置页做了就行”，而是配置页本身要决定系统能不能进入正常使用。建议把放行条件固定为产品规则，而不是人为约定。

必须通过 1 豆包 API 已配置并测试通过，因为日常解析主链依赖它。
必须通过 2 Serper API 已配置并测试通过，因为发现层默认依赖它。
必须通过 3 至少一个高价值深抓或校验能力已通过，例如 Firecrawl 或 Tavily。
建议项 Exa 和 Gemini 可以不是首日必需，但页面上仍要能配置与后续启用。
