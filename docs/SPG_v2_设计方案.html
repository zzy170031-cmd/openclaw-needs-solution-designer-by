系统定位

SPG 是一个面向游戏行业的多平台内容信号采集、清洗、归并与追踪系统。第一版聚焦中国 SLG（策略类）游戏赛道，覆盖从内容采集到热点拆解的全链路。

核心能力：通过多 API 调用链，将 TapTap / B站 / 微博 / 抖音 / 小红书 / 官方源的海量内容归并为可追踪的游戏信号，实现版本更新、赛季轮换、联盟争端、联动活动等事件的实时热点追踪与可视化。

API 调用链路示例

// ===== 日常采集流程 (每日自动执行) =====

// Step 1: Serper 搜索各 SLG 游戏最新动态
for game in ["率土之滨", "三国志战略版", "万国觉醒"] {
    let results = serper.search(format!("{game} 最新动态"), limit=10);  // ~10次/游戏
    let results2 = serper.search(format!("{game} 版本 赛季"), limit=10); // ~10次/游戏
    raw_items.extend(results);
}

// Step 2: Exa 深度语义搜索 (补充 Serper 未覆盖的长尾内容)
for game in ["率土之滨", "三战"] {
    let deep = exa.search_similar(
        format!("{game} 赛季攻略 阵容推荐"),
        num_results=5
    );  // ~5次/游戏
    raw_items.extend(deep);
}

// Step 3: Firecrawl 抓取高价值官方页面 (仅有变化时抓取)
let official_pages = [
    "https://stzb.163.com/news",
    "https://sango.xd.com/news"
];
for page in official_pages {
    if has_changed_since_last_crawl(page) {
        let content = firecrawl.scrape(page);  // ~2次/页
        raw_items.extend(parse_official(content));
    }
}

// Step 4: 全部原始内容注入 Pipeline
pipeline.ingest(raw_items);

// Step 5: 豆包 AI 分析 (无限调用，用于每条信号)
for signal in pipeline.unresolved_signals() {
    let analysis = doubao.analyze(signal.text, prompt="识别这是哪款SLG游戏的什么事件");
    pipeline.update_resolution(signal.id, analysis);
}

// Step 6: 低置信度信号用 Gemini 复核
for signal in pipeline.low_confidence_signals(threshold=0.65) {
    let review = gemini.analyze(signal.text, context=signal.trace);
    pipeline.update_resolution(signal.id, review);
}

// Step 7: Tavily 事件交叉验证 (对高价值事件)
for event in pipeline.new_events(min_signals=3) {
    let verification = tavily.search(format!("{} {} 确认", event.game, event.summary));
    pipeline.verify_event(event.id, verification);
}

每个阶段实现 PipelineStage trait，可独立测试、可插拔替换：

pub trait PipelineStage {
    type Input;
    type Output;
    type Error: std::error::Error;

    fn name(&self) -> &str;
    fn process(&self, input: Self::Input, ctx: &mut PipelineContext) -> Result<Self::Output, Self::Error>;
}

// 编排层 (~100 行，替代当前 781 行的 pipeline.rs)
pub fn run_pipeline(items: Vec<RawSourceItem>, ctx: &mut PipelineContext) -> Result<Vec<SourceSignal>> {
    items.into_iter().map(|raw| {
        let clean   = CleanStage::new().process(raw, ctx)?;
        let cand    = CandidateStage::new(&ctx.assets).process(&clean, ctx)?;
        let dedup   = DedupeStage::new().process((&clean, &cand), ctx)?;
        let resolve = ResolutionStage::new().process((&cand, &dedup), ctx)?;
        let event   = EventStage::new(&ctx.assets).process((&clean, resolve.game_id.as_ref()), ctx)?;
        Ok(build_signal(clean, cand, dedup, resolve, event, ctx))
    }).collect()
}

context_gap0.15上下文裁决分差

v0.1.2 问题: URL seed 子串匹配 [P0 修复]

[BUG] 当前 pipeline.rs:410 使用 contains(seed) 纯子串匹配

如果 seed 过短（如 "com"），会匹配所有 .com 域名的 URL，导致错误归并。

修复: 改为域名感知匹配，利用已有 url crate 解析域名后进行域名层级比对。

6. 去重策略

内容近重复

content_near_duplicate
  • 相同 URL 指纹
  • 正文引用链接匹配（转载/搬运）
  • 内容指纹完全相同

处理: 优先折叠，只保留 canonical

信息重复

information_duplicate
  • 同一游戏 + 同一事件 + 语义相似
  • 保留版本号/赛季号区分
  • 带时间窗口 (72h)

处理: 不删除，但不等权叠加热度

跨批次去重 新增

cross_batch
  • 指纹持久化到 SQLite
  • 每批先查库再查内存
  • 支持 append 模式累积

解决: 当前每批重置索引的问题

information_key 构成

information_key = "info::{scope_basis}::{event_key}::{semantic_fingerprint}::{time_window}"

scope_basis:
  已归并游戏 → "game::game_stzb"
  赛道降级   → "track::slg"
  未解析     → "source::raw_xxx"

event_key:   "season" / "version" / "commander" / ... / "general"

time_window: 每72小时一个窗口桶 (可配置)

10. 错误处理层次

// src/error.rs — 用 thiserror 定义结构化错误

#[derive(Debug, thiserror::Error)]
pub enum SpgError {
    #[error("asset loading failed: {0}")]
    AssetLoad(#[from] AssetError),

    #[error("input validation: {0}")]
    InputValidation(#[from] ValidationError),

    #[error("pipeline stage '{stage}' failed: {source}")]
    StageFailure { stage: String, source: Box<dyn Error + Send + Sync> },

    #[error("storage error: {0}")]
    Storage(#[from] StorageError),
}

#[derive(Debug, thiserror::Error)]
pub enum AssetError {
    #[error("url seed '{seed}' too broad (len < 8 or no dot)")]
    BroadUrlSeed { seed: String },

    #[error("duplicate game_id '{0}'")]
    DuplicateGameId(String),

    #[error("official account conflict: '{account}' maps to {owners:?}")]
    SharedOfficialAccount { account: String, owners: Vec<String> },
}
